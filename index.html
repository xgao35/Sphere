Sphere

Overview

Sphere is a startup that aims to become a decentralized corporate university that improves employees' functional skills by providing them with 3-week livestream courses instructed by field experts. In my UI/UX course, each team is tasked to iteratively design a piece of user experience for a startup's product, without seeing their actual product. With a passion for education technology, my team chose Sphere.

  • Project Type: course team project
  • Time: Oct 2022 - Nov 2022
  • Role: Designer

Based on Sphere's mission statement, we summed up the three most important aspects of their product:

Livestream courses

Organization of material

Networking

Sketching & Wireframing

Brainstorm Sketching

To start off, each of our four group members did 3 sketches of 3 main screens for the app:

  1. Home page (left column)
  2. Course overview page (middle column)
  3. Networking/chat page (right column)

Through discussion, we decided to keep these design choices to target Sphere's mission statement:

Livestream Courses

  1. Notify users through the homepage when courses are livestreaming
    1. So we can give users easy access to the livestream page and maximize user attendence to livestreams
  2. Provide users with live transcript and chat during livestream
    1. So livestream content can be more accessible, and users can discuss with instructor and classmates

Organization of Material

  1. Organize lectures by recency on the homepage and by livestream date on course overview page
    1. So users can access their last viewed lectures with ease, and they can find other lectures in an organized manner
  2. Provide users with a clear list of deadline and assignments in progress both on the homepage and in individual course overview pages
    1. So users can easily track their assignment progress and organize their work efficiently
  3. Use progress bars to quantify progress when possible

Networking

  1. Provide information on chat page about contact's occupation, employer, and common courses (if applicable)
    1. So users can engage in course-related conversations easily

Addressing Peer Feedback

We presented the first iteration of our propotype to peers, who gave us feedback after navigating through the prototype.

Here is a summary of the feedback that we received, and what we did to address these feedback.

Homepage

  • The chat-related notification looks too similar to course related notification. Maybe they should have different colors?
    • Move message notification to the nav bar, so only course related notifications show up on the homepage
  • The wording of "2/3 In progress" is too vague, since users don't know if it is referring to total number of courses or assignments
    • Change the wording to "1/3 courses completed"
    • Add shadows to make it clearer that the dark purple signifies the completed portion
  • Rather than a list of things in progress, the user would more likely want to directly see a list of due dates
    • Replace the "In Progress" section with "Today", which provides a list of due dates

Navigation Bar

  • It is a bit unclear that the second icon leads to "my courses"
    • Change the icon to an open book
  • Too much purple in the nav bar might distract attention from the main page
    • Restyle the nav bar background color to white and use icon color to indicate selection

Course Page

  • It can be very convenient if the user can directly enter into the course livestream from the course page
    • Add "Class is Streaming. Join now!" button that appears when the course is streaming
  • The purple background of the overview blurb is similar to clickable buttons, which can be misleading for the users
    • Change the background of the overview blurb to grey to make it look less interactable
  • The "Assignments" section can be made clearer by creating more distinction between items that are "not started" and "in progress
    • Since it can be messy to define "not started" and "in progress", we combine these two conditions into "incomplete"
  • The chat page contains chats with classmates, but where can users start chat with a classmate?
    • Add classmate profiles to the end of the course page

Chat Pages

  • The users might want to be able to see who are instructors and who are students from the chat pages
    • Add star next to instructor names in both the chat page and course page
    • Add "student" or "instructor" under the person's name in the dialogue page
  • It can be easier for the users to conduct course-related conversations if the they can access the course page directly from the chat page
    • Add link to course page from the "start a conversation" prompt

User Testing

We submitted our revised prototype to UserTesting.com and received feedback from 3 testers. We provided them with some instructions as well as post-test questions to get insight on their opinions.

Task instructions: Find out who the instructor for ENGN1010 is and send them a message.

User 1 Behavior Summary

  1. read over homepage
  2. noticed chat icon in nav bar and clicked to go to direct messages
  3. returned to homepage to find instructor
  4. clicked book icon to go to courses page
  5. clicked on ENGN1010 course to go to course overview page
  6. found the instructor!
  7. tried to click email option, then clicked direct message to open chat
  8. success! But the user was unsure if the prototype should allow them to actually send a message

User 2 Behavior Summary

  1. tried to click lecture under "Today" section on homepage
  2. clicked notification to join livestream
  3. tried to find instructor on livestream page
  4. returned to homepage and clicked on "Course Progress" graphic to go to courses page
  5. tried to find instructor listed under course name
  6. went to direct messages and clicked on the correct chat by chance
  7. returned to courses overview page and found instructor. Success!

User 3 Behavior Summary

  1. read over homepage
  2. clicked on book to go to my courses
  3. clicked on course overview page
  4. found instructor and noticed direct message button but chose to access direct messages through nav bar
  5. clicked on instructor name to go to correct chat. Success!

Post-test questions

  1. What frustrated you most about this app?
    1. User 1: I would say the lack of labels for the icons, which made navigation more exploratory rather than self-explanatory
    2. User 2: Lack of words under the four icons at the bottom of the screen
    3. User 3: Nothing, everything was easy to find and use
  2. Were there any interactive elements that were not immediately intuitive to you?
    1. User 1: No. I believe the icons at the bottom were the only confusing part to me
    2. User 2: All available functions worked well
    3. User 3: No, everything was self-explanatory. Anyone who is used to using a computer/smartphone should learn how to use this app pretty quickly
  3. What did you like about the app?
    1. User 1: The direct links to livestreams and direct messages
    2. User 2: The static menu on the bottom is really useful and prevents the user from being lost
    3. User 3: The simplicity of the design. I wish Canvas is like this

Overall, all three users completed the task, and they find the app easy to navigate.

Addressing User Feedback

  • One user was confused about the company name listed under the courses in my courses page. They expect to see instructor name rather than their own company name
    • List instructor name under courses instead
  • All users were confused by the lack of labels in the navigation bar to varying degrees
    • Add labels to navigation bar
  • Even though the star icon appears next to the instructor in the course overview page to signal its meaning, users do not intuitively interprete it as "instructor", and one user thought it means a favorited contact
    • Get rid of the star and clearly label instructors with "instr." instead
  • Users who directly entered the livestream from the homepage was a bit lost because there is no link between the livestream page to the course overview page
    • Add link that navigates from livestream to course overview page

Conclusions & Lessons

  1. Feedback is important in making sure that every aspect of the design is intentional. In this project, peer and user feedback for the first two mockups picked out many design choices that were not thoroughly considered from a user perspective. It is impossible to design completely without a bias towards what makes sense for oneself, so thinking iteratively with feedback along the way is extremely important.
  2. User logic can be very different from the designer's logic. The user testing videos showed me this principle in action for the first time. All of the users completed the assigned task from vastly different routes. For instance, to message the instructor, one user found the "direct message" button, but chose a more complicated route of navigating to the direct message page via finding the same icon in the navigation bar. To address this variety of user navigation logic, the designer must consider multiple routes to accomplish the same task.
  3. Information needs to be very obvious for users to learn. The design of the instructor star icon and navigation bar without labels were not intuitive for the users because the presentation of information is not immediately significant enough. People have a limited scope of attention, so to make the user's first encounter with the product is as easy as possible, they should not be expected to learn icons and navigations unless the instructions are exceedingly clear.
